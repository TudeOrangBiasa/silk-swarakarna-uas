# Triage Labels

The skills speak in terms of five canonical triage roles. This file maps those roles to the actual `Status:` line values used in this repo's local-markdown issue tracker.

| Role in skills | Status value in `Status:` line | Meaning |
| --- | --- | --- |
| `needs-triage` | `needs-triage` | Maintainer needs to evaluate this issue |
| `needs-info` | `needs-info` | Waiting on reporter for more information |
| `ready-for-dev` | `ready-for-dev` | Fully specified, ready for a developer to pick up |
| `ready-for-human` | `ready-for-human` | Requires human implementation |
| `wontfix` | `wontfix` | Will not be actioned |

When a skill mentions a role (e.g. "apply the AFK-ready triage status"), write the corresponding value from this table as the `Status:` line in the issue file.

Edit the right-hand column to match whatever vocabulary you actually use.
